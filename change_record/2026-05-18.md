## 菜品图片 AI 生成（像素风）

### 修改文件

| 文件 | 改动 |
|------|------|
| `ai/AIService.java` | `generateRecipeImage(String dishName, ...)` → `generateRecipeImage(RecipeItem recipe, ...)` |
| `ai/DashScopeService.java` | 实现阿里云通义万相 (qwen-image-2.0-pro) 多模态图片生成：构建像素风提示词（含食材清单+烹饪方式），120s 读超时，返回图片 URL |
| `ai/OpenAIService.java` | 适配新接口签名，使用 `recipe.getName()` |
| `ui/detail/RecipeDetailActivity.java` | 新增串行请求所有菜品图片；列表项和详情页均用 Glide 加载图片；占位图期间保留 `@drawable/apic` |

### 功能逻辑

- **触发时机**：菜品列表展示后立即串行请求图片
- **提示词结构**：菜品名 + 食材清单(含状态) + 烹饪方式 + 像素风要求（泰拉瑞亚风格、2000 像素点、PNG 透明底）
- **串行策略**：`Executors.newSingleThreadExecutor()` + `CountDownLatch` 真正等待每张完成再发下一张，避免阿里云 429 限流
- **展示流程**：占位图 → AI 图片 URL 返回 → Glide 加载 → 渐入替换
- **错误处理**：单张图片生成失败不影响其他菜品，保留占位图
- **资源释放**：`onDestroy()` 中关闭线程池

### 踩坑记录

| 问题 | 原因 | 修复 |
|------|------|------|
| HTTP 400 URL error | 使用旧 `wanx-v1` endpoint 格式 | 改为 `multimodal-generation/generation` + `qwen-image-2.0-pro` |
| 解析失败（即使 200） | API 返回字段是 `"image"` 而非 `"image_url"` | 解析时检查 `"image"` 字段 |
| 429 限流 | `generateRecipeImage` 内部又起 `new Thread()`，`Thread.sleep(1500)` 不能阻止线程堆积 | 改用 `CountDownLatch` 阻塞等待回调完成 |
| 429 仍存在 | 每分钟请求数超限（非并发限制） | 在 `DashScopeService.callImageGeneration()` 中加 429 自动重试（指数退避 5s→10s→20s），且每张图片完成后额外等待 2s |
| HTTP 同步图片生成慢且易 429 | 同步接口并发占用配额 | 改为异步 `text2image/image-synthesis` 接口（`qwen-image-plus`），创建任务后轮询结果，创建任务瞬间完成不阻塞 |

## 图片本地持久化存储

### 修改文件

| 文件 | 改动 |
|------|------|
| `util/ImageFileManager.java` | **新增** — 从 URL 下载图片保存到 `files/recipe_images/{recipeKey}.png`，提供 getLocalPath/saveImageFromUrl/deleteImage 方法 |
| `data/db/dao/RecipeDao.java` | 新增 `updateImagePath(recipeKey, imagePath)` 查询 |
| `ui/detail/RecipeDetailActivity.java` | 图片生成成功后通过 `ImageFileManager` 保存到本地，并异步更新 Room DB 的 image_path；`showDetail()` 优先从本地文件加载，其次 URL，最后占位图兜底 |
| `ui/profile/HistoryActivity.java` | 列表项使用 Glide 加载 `recipe.getImagePath()`，有则显示图片，无则占位图 |
| `ui/profile/FavoritesActivity.java` | 同上；ViewHolder 新增 `ivImage` 字段及绑定 |

### 功能逻辑

- **保存时机**：AI 图片生成成功回调中立即下载到本地
- **存储位置**：`data/data/{包名}/files/recipe_images/{recipeKey}.png`
- **路径映射**：优先本地路径，Glide 自动适配 `file://` 和 URL
- **历史/收藏回顾**：从 DB 读取 `image_path` → Glide 加载 → 显示图片
- **兜底策略**：imagePath 为空或文件不存在时显示 `@drawable/apic` 占位图
