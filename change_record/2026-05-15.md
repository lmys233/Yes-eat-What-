## 数据库种子数据修复

### 问题

数据库版本升至 2 后，破坏性迁移重建了表，但 `SeedCallback` 在 `onCreate` 中通过 `instance` 访问 DAO 时，`instance` 仍为 null（`build()` 未返回），导致 80 种食材和 8 个分类的种子数据从未写入数据库，食材选择页一片空白。

### 修改文件

| 文件 | 改动 |
|------|------|
| `data/db/AppDatabase.java` | 新增 `ensureSeeded()`，在 `build()` 完成后检测分类表是否为空，为空则在后台线程重新写入种子数据，同时删除在 `onCreate` 中引用 `instance` 的错误逻辑 |

## 收藏功能改用 History 表 flag 模式

### 改动说明

废弃 `favorite` 表的增删模式，改为在 `history` 表上标记 `is_favorite` 和 `favorited_at` 字段。收藏/取消收藏只改 flag，不删数据。

### 新增文件

| 文件 | 说明 |
|------|------|
| `ui/profile/FavoritesActivity.java` | 收藏列表页，按收藏时间倒序展示 |
| `res/layout/activity_favorites.xml` | 收藏列表布局（Toolbar + RecyclerView + 空状态） |

### 修改文件

| 文件 | 改动 |
|------|------|
| `data/db/entity/History.java` | 新增 `is_favorite`（boolean）和 `favorited_at`（long）字段 |
| `data/db/dao/HistoryDao.java` | 新增 `getFavorites()`（关联 GeneratedRecipe）、`updateFavoriteStatus()` |
| `data/repository/HistoryRepository.java` | 新增 `toggleFavorite()`（切换 flag + 回调）、`getFavoritesAsync()`、`checkFavorite()` |
| `ui/detail/RecipeDetailActivity.java` | 移除直接使用 `Favorite` 实体，改为 `HistoryRepository.toggleFavorite()` / `checkFavorite()` |
| `ui/profile/ProfileFragment.java` | "我的收藏"点击跳转 `FavoritesActivity` |
| `AndroidManifest.xml` | 注册 `FavoritesActivity` |
| `data/db/AppDatabase.java` | 版本 2 → 3 |

### 功能逻辑

- **收藏**：点击爱心 → history 表 `is_favorite=1, favorited_at=now` → 空心变实心主题色
- **取消收藏**：再次点击 → `is_favorite=0, favorited_at=0` → 实心变回空心
- **显示**：进入详情页 → `checkFavorite()` 查 history 表 flag → 决定爱心样式
- **收藏列表**：个人中心 → 我的收藏 → `getFavoritesAsync()` 查 `is_favorite=1` 的记录 → 按 `favorited_at DESC` 排序
- **数据保留**：取消收藏不删数据，flag 留着后续可用于推荐算法
- **关联**：通过 `HistoryWithRecipe` 关联 `GeneratedRecipe`，复用浏览记录结构
