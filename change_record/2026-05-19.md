### 独立生成工具说明

**位置**: `tools/ingredient-image-generator/IngredientImageGenerator.java`

**功能**: 遍历 80 种预制食材（与 DataSeeder 一致，id=1~80），串行调用 AI 生成像素风图标图片

**运行方式**:
```bash
cd tools/ingredient-image-generator
javac IngredientImageGenerator.java
java IngredientImageGenerator --api-key=your_key --output=./output
```

**输出**: `ingredient_1.png` ~ `ingredient_80.png`（文件名 = 食材 ID）

**使用流程**:
1. 运行生成工具，得到 80 张 PNG 图片
2. 将图片复制到 `app/src/main/assets/ingredient_images/` 目录
3. 重新编译 APP，图片被打包到 APK
4. APP 运行时通过 `IngredientItem.id` 查找 `ingredient_{id}.png` 并显示

### 映射关系

- `SystemIngredient.id` (1~80) ↔ `ingredient_{id}.png` ↔ `IngredientItem.id`
- `IngredientAdapter` 检查 `IngredientItem.imagePath` → 有则 Glide 加载，无则 `@drawable/apic` 占位图
- 优先级：本地 `files/ingredient_images/` → 占位图

## 修复：新增食材生成图片不显示

### 问题

自定义食材 AI 生成图片成功后，图片没有显示在预览框和食材列表中。

### 根因

| # | 问题 | 原因 |
|---|------|------|
| 1 | `NetworkOnMainThreadException` | `onSuccess` 回调中 `saveImageFromUrl()`（OkHttp 同步下载）被包在 `runOnUiThread` 里，在主线程做网络请求抛异常，下载失败返回 null |
| 2 | 数据库回读图片路径丢失 | `IngredientRepository.getIngredients()` 加载 `CustomIngredient` 时，图片路径存在 `icon` 字段，但适配器检查的是 `getImagePath()`，未同步导致显示占位图 |
| 3 | Fragment 分离时崩溃风险 | 后台线程直接调用 `requireActivity()`/`requireContext()`，若用户提前返回则抛 `IllegalStateException` |

### 修改文件

| 文件 | 改动 |
|------|------|
| `ui/order/OrderFragment.java` | 将 `saveImageFromUrl()` 移出 `runOnUiThread`，在后台线程下载完成后再切回主线程更新 UI；添加 `isAdded()` 守卫 |
| `data/repository/IngredientRepository.java` | 加载自定义食材后调用 `setImagePath(ci.getIcon())` 同步图片路径 |

## 修复：自定义食材图片显示为其他食材图片

### 问题

自定义食材添加到列表后，显示的图片不是 AI 生成的图片，而是某个已存在系统食材的图片（如猪五花肉）。

### 根因

`IngredientAdapter` 的图片加载逻辑是：先根据 `ingredientId` 在 drawable 中查找 `ingredient_{id}.png`，找不到才 fallback 到 `imagePath`。

自定义食材存入 Room 数据库后获得自增 ID（1, 2, 3...），与系统食材的 drawable 文件名 `ingredient_1.png`～`ingredient_80.png` 冲突。适配器先找到了 drawable 资源就直接加载了，跳过了 `imagePath`。

### 修改

| 文件 | 改动 |
|------|------|
| `adapter/IngredientAdapter.java` | 自定义食材优先判断 `isCustom()`，直接走 `imagePath` 加载，跳过 drawable 查找；系统食材保留原有 drawable 查找逻辑 |

## 变更：食材图片文件名改为 UUID

### 原因

之前用食材中文名 sanitize 后做文件名，中文全变下划线容易冲突。改用 UUID 保证唯一。

### 修改

| 文件 | 改动 |
|------|------|
| `util/IngredientImageHelper.java` | `saveImageFromUrl()` 改用 `UUID.randomUUID()` 生成文件名，去掉无用的 `getLocalPath`/`isGenerated`/`deleteImage` 等方法 |
| `ui/order/OrderFragment.java` | 同步更新 `saveImageFromUrl` 调用（去掉已移除的 `name` 参数） |
